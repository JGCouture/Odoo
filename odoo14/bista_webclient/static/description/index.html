Force apply 'AND' condition for search view.

                
    e.g.

    <search string="Documents">
        ...
        <field name="name" string="Document (AND)" options="{'filter_domain_condition': 'and'}"/>
        <field name="name" string="Document (OR)"/>
        ...
    </search>
Developer can use 'AND', 'and' and '&' as value of filter_domain_condition.

On drag open record on kanban view

Options

Option Value Description
form_dialog_size 'small', 'medium' or 'large' Specify the popup size. Default value is 'medium'.
drag_forward_domain e.g. [('art_state', '=', False), ...N] Popup will open based on specified domain else it will work as it is.
drag_backward_domain e.g. [('art_state', '=', False), ...N] Popup will open based on specified domain else it will work as it is.
forward_form_view_ref Needs to specify the value of 'forward_form_view_ref' if user want to open specific form view when move forward kanban record. otherwise it will take default form view
backward_form_view_ref Needs to specify the value of 'forward_form_view_ref' if user want to open specific form view when move forward kanban record. otherwise it will take default form view 
                
    e.g.

    <record id="view_art_production_kanban" model="ir.ui.view">
        ...
        <field name="arch" type="xml">
            <kanban on_drag_open_record="{
                    'form_dialog_size': 'small',
                    'drag_forward_domain': [('art_state', '=', False)],
                    'drag_backward_domain': [],
                    'forward_form_view_ref': 'bista_art_work.art_production_quick_form_view',
                    'backward_form_view_ref': 'bista_art_work.art_production_quick_form_view',
                }">
                ...
                ...
            </kanban>
        </field>
        ...
    </record>
